# PIX Payments API

The PIX Payments API allows you to create, manage, and process PIX payments through multiple payment gateways. This section covers all PIX-related endpoints.

## Create PIX Payment

Send a PIX payment to a specific PIX key.

<RequestExample>
```bash cURL
curl -X POST "https://api.braix.com.br/requestPayment" \
  -H "Authorization: Bearer your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 10.50,
    "pixKey": "user@example.com",
    "webhookUrl": "https://your-domain.com/webhooks/braix",
    "externalId": "order_123",
    "gateway": "woovi",
    "description": "Payment for order #123"
  }'
```
</RequestExample>

<ResponseExample>
```json Success
{
  "success": true,
  "data": {
    "transactionId": "tx_123456789",
    "status": "pending",
    "amount": 10.50,
    "pixKey": "user@example.com",
    "externalId": "order_123",
    "gateway": "woovi",
    "createdAt": "2024-01-15T10:30:00Z"
  }
}
```
</ResponseExample>

### Parameters

<ParamField body="amount" type="number" required>
The payment amount in Brazilian Reais (BRL). Must be greater than 0.01.
</ParamField>

<ParamField body="pixKey" type="string" required>
The PIX key to send the payment to. Can be CPF, CNPJ, email, phone, or random key.
</ParamField>

<ParamField body="webhookUrl" type="string" required>
URL to receive webhook notifications for this payment.
</ParamField>

<ParamField body="externalId" type="string">
Your internal reference ID for this payment.
</ParamField>

<ParamField body="gateway" type="string" required>
The payment gateway to use. Options: `woovi`, `treeal`, `voluti`, `efi`, `playshark`, `pluggou`, `bancoe2`, `primepag`, `wepayments`, `xgate`, `xgate2`.
</ParamField>

<ParamField body="description" type="string">
Description of the payment for your records.
</ParamField>

<ParamField body="withdrawalType" type="string">
Type of withdrawal. Options: `qrCode`, `crypto`.
</ParamField>

## Generate PIX QR Code

Create a PIX QR code for receiving payments.

<RequestExample>
```bash cURL
curl -X POST "https://api.braix.com.br/requestQrCode" \
  -H "Authorization: Bearer your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 25.00,
    "webhookUrl": "https://your-domain.com/webhooks/braix",
    "externalId": "qr_123",
    "description": "QR Code payment"
  }'
```
</RequestExample>

<ResponseExample>
```json Success
{
  "success": true,
  "data": {
    "qrCodeId": "qr_123456789",
    "qrCode": "00020126580014br.gov.bcb.pix...",
    "qrCodeImage": "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA...",
    "amount": 25.00,
    "status": "active",
    "expiresAt": "2024-01-15T11:30:00Z"
  }
}
```
</ResponseExample>

### Parameters

<ParamField body="amount" type="number" required>
The payment amount in Brazilian Reais (BRL). Must be greater than 0.01.
</ParamField>

<ParamField body="webhookUrl" type="string">
URL to receive webhook notifications when payment is received.
</ParamField>

<ParamField body="externalId" type="string">
Your internal reference ID for this QR code.
</ParamField>

<ParamField body="description" type="string">
Description of the QR code payment.
</ParamField>

<ParamField body="splits" type="array">
Array of split payment objects for multi-party payments.
</ParamField>

<ParamField body="devedor" type="object">
Debtor information for the PIX payment.
</ParamField>

## Check Transaction Status

Get the current status of a PIX transaction.

<RequestExample>
```bash cURL
curl -X POST "https://api.braix.com.br/checkTransactionStatus" \
  -H "Authorization: Bearer your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "endToEndId": "E12345678202401151234567890",
    "txId": "tx_123456789"
  }'
```
</RequestExample>

<ResponseExample>
```json Success
{
  "success": true,
  "data": {
    "transactionId": "tx_123456789",
    "endToEndId": "E12345678202401151234567890",
    "status": "completed",
    "amount": 10.50,
    "pixKey": "user@example.com",
    "completedAt": "2024-01-15T10:35:00Z"
  }
}
```
</ResponseExample>

### Parameters

<ParamField body="endToEndId" type="string">
The end-to-end identifier of the PIX transaction.
</ParamField>

<ParamField body="txId" type="string">
The transaction ID from the payment gateway.
</ParamField>

<Note>
Either `endToEndId` or `txId` is required to check transaction status.
</Note>

## Process PIX Refund

Refund a completed PIX payment.

<RequestExample>
```bash cURL
curl -X PUT "https://api.braix.com.br/E12345678202401151234567890/devolucao" \
  -H "Authorization: Bearer your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "valor": 10.50
  }'
```
</RequestExample>

<ResponseExample>
```json Success
{
  "success": true,
  "data": {
    "refundId": "ref_123456789",
    "endToEndId": "E12345678202401151234567890",
    "amount": 10.50,
    "status": "processing",
    "requestedAt": "2024-01-15T10:40:00Z"
  }
}
```
</ResponseExample>

### Parameters

<ParamField path="endToEndId" type="string" required>
The end-to-end identifier of the PIX transaction to refund.
</ParamField>

<ParamField body="valor" type="number" required>
The refund amount in Brazilian Reais (BRL). Must be greater than 0.01.
</ParamField>

## List User Transactions

Get a list of transactions for the authenticated user.

<RequestExample>
```bash cURL
curl -X GET "https://api.braix.com.br/transactions?page=1&limit=20&status=completed" \
  -H "Authorization: Bearer your_api_key_here"
```
</RequestExample>

<ResponseExample>
```json Success
{
  "success": true,
  "data": {
    "transactions": [
      {
        "id": "tx_123456789",
        "amount": 10.50,
        "status": "completed",
        "pixKey": "user@example.com",
        "gateway": "woovi",
        "createdAt": "2024-01-15T10:30:00Z",
        "completedAt": "2024-01-15T10:35:00Z"
      }
    ],
    "pagination": {
      "page": 1,
      "limit": 20,
      "total": 1,
      "totalPages": 1
    }
  }
}
```
</ResponseExample>

### Query Parameters

<ParamField query="page" type="integer" default="1">
Page number for pagination.
</ParamField>

<ParamField query="limit" type="integer" default="20">
Number of transactions per page (max 100).
</ParamField>

<ParamField query="status" type="string">
Filter by transaction status: `pending`, `completed`, `failed`, `cancelled`.
</ParamField>

<ParamField query="gateway" type="string">
Filter by payment gateway.
</ParamField>

## Export PIX Data

Export PIX transaction data in various formats.

<RequestExample>
```bash cURL
curl -X POST "https://api.braix.com.br/export" \
  -H "Authorization: Bearer your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "pix_in",
    "start_date": "2024-01-01",
    "end_date": "2024-01-31",
    "format": "csv",
    "fields": "id,amount,status,created_at"
  }'
```
</RequestExample>

<ResponseExample>
```json Success
{
  "success": true,
  "message": "PIX data exported successfully",
  "data": {
    "exportId": "exp_123456789",
    "downloadUrl": "https://api.braix.com.br/exports/exp_123456789/download",
    "format": "csv",
    "recordCount": 150,
    "expiresAt": "2024-01-22T10:30:00Z"
  }
}
```
</ResponseExample>

### Parameters

<ParamField body="type" type="string" required>
Type of data to export: `pix_in`, `pix_out`, `both`.
</ParamField>

<ParamField body="start_date" type="string">
Start date for export in ISO 8601 format.
</ParamField>

<ParamField body="end_date" type="string">
End date for export in ISO 8601 format.
</ParamField>

<ParamField body="status" type="integer">
Filter by status: 0=pending, 1=completed, 2=failed, 3=cancelled.
</ParamField>

<ParamField body="min_amount" type="number">
Minimum transaction amount to include.
</ParamField>

<ParamField body="max_amount" type="number">
Maximum transaction amount to include.
</ParamField>

<ParamField body="format" type="string" default="json">
Export format: `json`, `csv`.
</ParamField>

<ParamField body="fields" type="string">
Comma-separated list of fields to include in export.
</ParamField>

<ParamField body="order_by" type="string">
Field to sort by: `created_at`, `amount`, `status`.
</ParamField>

<ParamField body="order_direction" type="string">
Sort direction: `asc`, `desc`.
</ParamField>
