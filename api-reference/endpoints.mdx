# API Endpoints

This section provides a complete reference for all available endpoints in the Braix API.

## Authentication

### POST /auth/token

Obtain an access token using your client credentials.

<RequestExample>
```bash cURL
curl -X POST "https://api.braix.com.br/auth/token" \
  -H "Authorization: Basic $(echo -n 'clientId:clientSecret' | base64)" \
  -H "Content-Type: application/json"
```
</RequestExample>

<ResponseExample>
```json Success
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600
}
```
</ResponseExample>

## PIX Operations

### POST /requestQrCode

Generate a PIX QR Code for payment collection.

<RequestExample>
```bash cURL
curl -X POST "https://api.braix.com.br/requestQrCode" \
  -H "Authorization: Bearer your_access_token" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 10,
    "webhookurl": "https://webhook-test.com/99fc09d1286656428b57d2ec2a520a98"
  }'
```
</RequestExample>

<ResponseExample>
```json Success
{
  "success": true,
  "qrCode": "00020126580014br.gov.bcb.pix...",
  "qrCodeImage": "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA...",
  "transactionId": "tx_123456789"
}
```
</ResponseExample>

### POST /requestQrCode (Split Payment)

Generate a PIX QR Code with split payment functionality.

<RequestExample>
```bash cURL
curl -X POST "https://api.braix.com.br/requestQrCode" \
  -H "Authorization: Bearer your_access_token" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 10,
    "devedor": {
      "cpf": "12345678901",
      "name": "Jo√£o Silva"
    },
    "splits": [
      {
        "username": "partner1",
        "percentageSplit": "50"
      }
    ],
    "webhookurl": "https://webhook-test.com/99fc09d1286656428b57d2ec2a520a98"
  }'
```
</RequestExample>

### POST /requestQrCode (Direct Payment)

Make a direct PIX payment to a specific PIX key.

<RequestExample>
```bash cURL
curl -X POST "https://api.braix.com.br/requestQrCode" \
  -H "Authorization: Bearer your_access_token" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 10,
    "pix_key": "user@example.com",
    "withdrawaltype": "email",
    "wallet": "user_wallet_123",
    "webhookurl": "https://webhook-test.com/99fc09d1286656428b57d2ec2a520a98"
  }'
```
</RequestExample>

### PUT /pix/{endToEndId}/devolucao

Refund a PIX transaction.

<RequestExample>
```bash cURL
curl -X PUT "https://api.braix.com.br/pix/E60701190202509291927DY5IHAXSHJX/devolucao" \
  -H "Authorization: Bearer your_access_token" \
  -H "Content-Type: application/json" \
  -d '{
    "endToEndId": "E60701190202509291927DY5IHAXSHJX",
    "valor": "1.0"
  }'
```
</RequestExample>

<ResponseExample>
```json Success
{
  "success": true,
  "refundId": "refund_123456789",
  "status": "processing"
}
```
</ResponseExample>

## Transaction Management

### POST /api/checkTransactionStatus

Check the status of a specific transaction.

<RequestExample>
```bash cURL
curl -X POST "https://api.braix.com.br/api/checkTransactionStatus" \
  -H "Authorization: Bearer your_access_token" \
  -H "Content-Type: application/json" \
  -d '{
    "endToEndId": "E12345678202407080309abc123",
    "txId": "transaction-id-123"
  }'
```
</RequestExample>

<ResponseExample>
```json Success
{
  "success": true,
  "status": "completed",
  "amount": 10.00,
  "timestamp": "2024-01-15T10:30:00Z"
}
```
</ResponseExample>

## Reports and Analytics

### POST /api/pix/export

Export PIX transaction reports in various formats.

<RequestExample>
```bash cURL
curl -X POST "https://api.braix.com.br/api/pix/export" \
  -H "Authorization: Bearer your_access_token" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "both",
    "format": "json",
    "page": 1,
    "perPage": 100,
    "startDate": "2024-01-01T00:00:00Z",
    "endDate": "2024-12-31T23:59:59Z",
    "status": 1,
    "gateway": "treeal"
  }'
```
</RequestExample>

<ResponseExample>
```json Success
{
  "success": true,
  "data": [
    {
      "id": "tx_123456789",
      "amount": 10.00,
      "status": "completed",
      "createdAt": "2024-01-15T10:30:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "perPage": 100,
    "total": 1
  }
}
```
</ResponseExample>

## Error Responses

All endpoints may return the following error responses:

<ResponseField name="400 Bad Request" type="object">
  Returned when the request parameters are invalid.

  <Expandable title="Error Response">
    ```json
    {
      "success": false,
      "message": "Invalid parameters",
      "error": "Validation failed"
    }
    ```
  </Expandable>
</ResponseField>

<ResponseField name="401 Unauthorized" type="object">
  Returned when authentication fails.

  <Expandable title="Error Response">
    ```json
    {
      "success": false,
      "message": "Unauthorized",
      "error": "Invalid or expired token"
    }
    ```
  </Expandable>
</ResponseField>

<ResponseField name="404 Not Found" type="object">
  Returned when the requested resource is not found.

  <Expandable title="Error Response">
    ```json
    {
      "success": false,
      "message": "Not Found",
      "error": "Transaction not found"
    }
    ```
  </Expandable>
</ResponseField>

<ResponseField name="500 Internal Server Error" type="object">
  Returned when an internal server error occurs.

  <Expandable title="Error Response">
    ```json
    {
      "success": false,
      "message": "Internal Server Error",
      "error": "An unexpected error occurred"
    }
    ```
  </Expandable>
</ResponseField>
